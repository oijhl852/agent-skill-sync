# Seedance 分镜生成器

专业的AI脚本和分镜生成系统，用于在Seedance 2.0平台上创建专业的AI视频系列。

## 平台参数

| 维度 | 规格 |
|------|------|
| 图片输入 | jpeg/png/webp，≤9张，单张<30MB |
| 视频输入 | mp4/mov，≤3个，总时长2-15s，分辨率480p-720p |
| 音频输入 | mp3/wav，≤3个，总时长≤15s |
| 混合上限 | 最多12个文件（图片+视频+音频合计） |
| 生成时长 | 4-15秒，可自由选择 |
| 分辨率 | 支持2K输出 |
| @引用 | `@图片1`~`@图片9`、`@视频1`~`@视频3`、`@音频1`~`@音频3` |

### ⚠️ 限制

- 不支持上传含有写实真人脸部的素材，系统自动拦截
- 有参考视频时生成消耗更多
- 视频延长时，生成长度应选"新增部分"的时长

## Workflow

按照以下顺序流程将源材料转换为可用于制作的视频脚本和分镜提示词：

### 1. 分析输入

**确定输入类型：**
- **完整文本**：需要改编和分集的完整小说/文章 → 走完整流程
- **大纲**：需要完整脚本开发的简短概念 → 走完整流程
- **单个镜头/场景描述**：已有具体画面描述 → 跳到 Step 5 直接出分镜提示词

**提取核心元素：** 主角和关键角色、核心冲突和叙事弧线、场景/世界观、关键情节和情感节拍、核心梗/卖点

如果输入模糊或不完整，请提出澄清问题。

### 2. 确认制作参数

1. **视觉风格**：写实/动画/水墨/科幻/复古/电影感/其他
2. **时长**：短剧5集×15s≈75秒；长剧20集×15s≈5分钟
3. **画幅比例**：16:9横屏 / 9:16竖屏 / 2.35:1电影宽屏
4. **基调**：史诗/温馨/悬疑/欢快/忧伤
5. **核心梗**：绝境反杀/复仇爽剧/治愈温馨/悬疑惊悚等

**场景参数速查**（根据用户场景，自动匹配合适的布光和色调）：

| 场景 | 布光方案 | 色调 |
|------|---------|------|
| 现代家居白天 | 窗外自然光为主，白色墙面反射补亮 | 自然日光色温微暖 |
| 现代家居夜晚 | 台灯/落地灯单弱光源，大范围暗部 | 暖暗调 |
| 酒店房间 | 窗帘半掩柔光，暖色床头灯补光 | 暖色调低对比 |
| 派出所/办公 | 顶部日光灯均匀照明，无额外补光 | 冷白调 |
| 高档餐厅 | 暖色环境光均匀，金色装饰反射 | 暖金色调 |
| 雪地外景 | 阴天漫射光，雪面反光底部补亮 | 冷白/青灰调 |
| 古装室内 | 灶台火光+窗外天光混合 | 暖黄橘色调 |
| 黄昏户外 | 天光+人工灯混合 | 冷暖光混搭 |
| 非洲夜外 | 月光为主，侧逆冷色勾勒轮廓 | 月光冷白偏蓝 |
| 深夜大宅 | 室内单一弱光源，明暗交替 | 暖暗调+暗角 |

### 3. 生成完整剧本结构

**重要：输出必须严格遵循以下格式。**

#### 3.1 故事分析模板

```
# [Title] - 剧本

一、核心梗
[两字短语，如：绝境反杀、复仇爽剧]

二、故事梗概
故事背景：[时间、地点、初始状态]
开场冲突：[什么打破现状]
主角画像：[核心特质、当前状态]
主线事件：[推进→高潮→解决]
结局：[最终结果、角色转变]

三、一句话卖点
[情绪号召+核心冲突+视觉高潮，如：忍无可忍无须再忍！风雪夜林冲长枪出鞘血溅山神庙！]

四、人物小传
[每个主要角色：]

**角色名**
视觉形象：[年龄、外貌、服装、标志性道具]
身份背景：[社会身份、与主角关系、过往经历]
核心标签：[2-4个特质，如：隐忍不发、悲情英雄、杀神降临]
性格特点：[正常状态→压力下表现→爆发时]
金句：[最能代表角色的台词]

五、剧本大纲
前期（起）：[建立世界、主角现状、触发事件]
中期（承/转）：[冲突发展、转折点]
后期（高潮）：[最终对决、情感爆发]
尾声（收尾）：[结局状态、主题升华、余韵]

六、剧本正文
```

#### 3.2 分镜头剧本格式

**每个镜头以 `△ ` 开头：**

```
第X集
X-X [日/夜] [内/外] [场景名称]
道具：[道具列表]
出场人物：[角色列表]

△ 【空镜/开场镜头】[运镜+画面+光影+氛围]
△ [镜头2：运镜+动作+细节]
角色名（os）：[内心独白/旁白]
△ [反应镜头]
角色名（对白/动作描述）：[带情绪的对白]
△ [动作序列用 → 连接：A → B → C]
【闪回】[回忆镜头内容]
【闪回结束】
△ [回到现实，情绪反应]
角色名（vo）：[画外音，人物不在画面]
△ [高潮动作镜头]
△ 【空镜】[结尾氛围镜头]
【字幕：xxx】
```

**格式规则：**

1. **每个镜头以 `△ ` 开头**（triangle + space）
2. **镜头语言必须具体：**
   - 景别：远景/全景/中景/近景/特写/大特写
   - 运镜：推镜头/拉镜头/摇镜头/移镜头/跟镜头/环绕镜头/升降镜头/希区柯克变焦/一镜到底/手持晃动
   - 角度：平视/俯拍/仰拍/低角度/鸟瞰/鱼眼镜头/第一人称视角
   - 节奏：慢动作/快切/延时/升格/硬切/卡点/定格慢放
   - 组合示例：`中景推近`、`特写环绕`、`远景俯拍`、`快速推近`、`缓慢拉远`
   - 焦点：浅景深/深景深/焦点转移/虚化背景/对焦模糊

3. **对白标注：**
   - `角色名（os）` — 画外音/内心独白
   - `角色名（vo）` — 画外音，人物不在画面
   - `角色名（惊/怒/喜/怯/冷笑）` — 带情绪的对白
   - `角色名` — 普通对白

4. **特殊元素：**
   - `【空镜】` — 氛围镜头，无人物
   - `【闪回】` / `【闪回结束】` — 回忆段落
   - `【字幕：xxx】` — 需要的屏幕文字（否则标注"无字幕"）

5. **动作序列：** 使用 `→` 连接连续动作
6. **感官细节：** 视觉（颜色/光影/构图）+ 听觉（声音）+ 触觉（温度/质感）

### 4. 创建资产生成计划

**对所有视觉资产进行分类和编号：**

| 类别 | 前缀 | 示例 | 描述 |
|------|------|------|------|
| 角色 | C01-C99 | C01 林冲·正面全身 | 每个角色多个角度 |
| 场景 | S01-S99 | S01 草料场·雪景 | 关键位置 |
| 道具 | P01-P99 | P01 长枪 | 重要物品 |

**资产生成提示词格式：**
```
### [编号] — [名称]
[风格前缀]，[详细视觉描述]，[技术规格]
```

**风格前缀示例：**
- Chinese ink wash painting style mixed with anime cel-shading
- Cinematic photorealistic style with dramatic lighting
- 3D Pixar-style animation rendering
- Sci-fi cyberpunk aesthetic with neon lighting

**注意：** 角色参考图风格必须与视频主题一致：仙侠→3D国漫、古风→水墨/工笔、科幻→CG概念设计、现实→电影写实摄影、美食→商业摄影、动漫→对应美术风格。

### 5. 生成 Seedance 2.0 分镜提示词

#### 5.1 素材上传列表

```
| 上传位置 | 素材ID | 素材描述 |
| 图片1 | C01 | 角色参考 |
| 图片2 | S01 | 场景参考 |
```

#### 5.2 分镜提示词（两种格式可选）

**格式A — 结构化标签式**（适合单个场景/重氛围）：

```
【风格】[写实/古装/动漫风格]，[画质]，[氛围]
【镜头】ARRI MINI拍摄，中焦段85mm，大光圈f/1.2浅景深
【环境】[场景详细描述]
【布光】[光源/色温/补光说明]
【光影】[逆光/侧逆光/伦勃朗光等]
【色调】[主色调/对比度]
【特效】[虚化/颗粒/CG特效等]
【运镜】[手持/跟拍/固定等]
【表演】[动作要求/表情控制/眨眼]
【字幕】无字幕
【音频】仅环境音（[具体环境音描述]）

—————— 剧情动作线 ——————
[具体分镜动作用时间流描述]
```

**格式B — 时间轴式**（适合15秒完整叙事）：

```
[风格描述]，15秒，9:16竖屏，[整体氛围]

0-3秒：[场景建立] — [运镜]，[画面描述]，[氛围]
3-7秒：[主体引入/发展] — [运镜]，[动作]，[情绪]
7-12秒：[高潮/冲突] — [运镜]，[情绪爆发]，[视觉冲击]
12-15秒：[收尾/落版] — [运镜]，[最终画面]，[余韵]

【声音】[配乐风格] + [音效] + [对白/旁白]
【参考】@图片1 [用途]，@图片2 [用途]
```

**@引用规范：**
- `@图片1 作为首帧` / `@图片2 作为角色参考` / `@图片3-5 场景参考`
- `@视频1 参考运镜` / `@视频2 参考动作节奏`
- `@音频1 用于配乐` / `@音频2 用于对白音色`
- 素材多时必须检查各@对象不要搞混

**集间衔接 (Ep 2+):** `将@视频1延长15s`

**集间衔接**：记录每段最后一秒的画面状态（主体/背景/光线/构图），确保下一段开场能无缝接上，画面状态完全一致。

#### 5.3 提示词核心技巧

1. **写物理动作，不写心理状态**：Seedance 对具体可执行动作词响应最好
   - ❌ "一身正气"
   - ✅ "偏头避开视线，面色恢复平静"
   
2. **微表情必须拆解为显性动作**：
   - "压抑狂喜" → "嘴角短暂上扬即收，喉结微动"
   - "松了一口气" → "紧绷肩膀猛然一沉垮下，眉头松开低头缓一口气"
   - "心生一计" → "眼底精光一闪，嘴角勾起狡黠笑容"

3. **事件链写成一个不可拆分的连续动作**：
   - "视线短暂落在对方身上 → 随即偏头转向一侧 → 表情恢复平静"

4. **用速度副词替代情绪词**：`猛然`、`瞬间`、`快速` 比"恐惧""愤怒"有效

5. **对白节奏控制**：`语速自然平缓，断句分明，句尾有明显停顿`、`勿抢拍勿拖拍`

6. **情绪类反应镜头视线必须朝向镜头**：单人的情感反应镜头（恐惧、欣慰、如释重负、悲伤、担忧等）中，角色视线应"落向镜头"或"凝视镜头"，而不是看向画面中的环境或其他人物。这建立角色与观众的直接情感连接，让观众代入角色的内心感受。
   - ❌ "视线落向前方雪面"、"偏头看向身旁"
   - ✅ "视线怯怯落向镜头"、"眼底的恐慌透过镜头直抵人心"

#### 5.4 实战问题速查

| 常见问题 | 原因 | 修正方法 |
|---------|------|---------|
| **总是生成白天** | 布光写了"自然光模拟+暖色调" | 把光源改成具体夜晚光源（月光/台灯/烛光），色调加"冷夜色基调" |
| **金碧辉煌餐厅变成暗光** | 写了"暗光氛围""仅局部受光" | 改成"暖色环境光均匀铺满，金色装饰反射光，水晶吊灯散光" |
| **台词收尾太急/忽快忽慢** | 没写节奏控制 | 加独立`【对话】`标签：语速自然平缓、断句分明、句尾停顿 |
| **抢夺/争斗没有拉扯感** | 静态词"护着""保护" | 改成动态："死死攥住往回拽""身体弓起被拖得前倾""布料绷紧抖动" |
| **"松口气"表情出不来** | 抽象情绪词 | 拆成身体动作："肩膀塌下→闭眼深呼吸→攥拳的手微松→身体软下来" |
| **多人对话没有反应镜头** | 所有台词连在一起 | 每隔4-6秒台词主动插入对方的无台词反应镜头（眼神/微表情/点头） |
| **角色被拖躺着的状态丢失** | 模型默认站立 | 每个关键动作前重复一次"躺在地上的XXX" |

## 脚本写作指南

### 15秒剧集节奏

| 剧集类型 | 镜头数量 | 节奏 | 说明 |
|---------|---------|------|------|
| 对话/情感 | **5-6个** | 每镜2.5-3.5秒 | 短剧实战验证的最佳密度，少于5镜拖沓，多于7镜急促 |
| 动作/战斗 | 5-7个 | 每镜2-3秒 | 密集动作可稍快 |
| 蒙太奇/序列 | 6-8个 | 每镜2秒 | 快速情绪推进 |

### 镜头分配原则（短剧实战总结）

| 原则 | 规范 |
|------|------|
| 主角屏占比 | 两个主要角色各占 **~40%**，合计 80% |
| 配角出镜 | 每段最多 **1 个镜头**，他们是陪衬不是双主演 |
| 反打/反应镜头 | 每 4-6 秒台词后必须插入对方的无台词反应镜头 |
| 避免乒乓切 | 同一角色不可连续切换镜头；禁止 A 左角度→A 右角度→A 左角度 等同人视角反复横跳，没有任何叙事意义 |
| 动作镜头 | 关键物理动作（扇巴掌、推搡、手势）给独立镜头，1-2 秒足够 |

### 对话镜头切换铁律（实战验证）

多人对话是 AI 生成视频最容易翻车的地方。以下铁律来自实战踩坑，写分镜时必须逐条核对：

**铁律 0：谁说话给谁镜头**

台词和画面必须同步。说话者的对白必须配上说话者的画面，不可在台词说到一半时将镜头切给静默的其他角色。这不是"反应镜头"，是叙事错位。

```
✅ 正确：A 说 → 拍 A → A 说完 → 切 B 反应 → B 说 → 拍 B
❌ 错误：A 正在说 → 画面切给 B（B 没说话，嘴没动）
```

**铁律 1：长台词必须穿插反应镜头**

当 A 的台词超过一段话时，必须在中途给 B 切一个无台词反应镜头——让观众看到听者的表情。这不仅是节奏问题，更是 Seedance 的物理约束：单人长时间说话容易变成"站桩输出"，画面死板。

```
✅ 正确写法：
A（对白）：[第一句话，≤8秒]
△ 【反应镜头·近景】B 的反应动作
A（对白）：[第二句话]

❌ 错误写法：
A（对白）：[一大段话从头说到尾，没有任何镜头切换]
```

**铁律 2：切镜头必须等话说完，不在句中打断**

镜头切换只能发生在**对白句与句之间**，绝对不能在一句话说到一半时切走。AI 不理解语言节奏——如果在 `"我告诉你……"` 后面立即切给 B，生成出来的效果就是说话者嘴还在动但画面已经跳走了（或更糟：嘴型对不上）。

```
✅ 在句号/感叹号/问号后切：
A（怒）: 你竟敢背叛我！
△ 【反应镜头】B 冷笑
B（冷笑）：背叛？你自己做了什么心里清楚。

❌ 在句中切：
A（怒）: 你竟敢……
△ 【反应镜头】B 冷笑
A（怒）：……背叛我！ ← 一句话被拆成两截，节奏崩坏
```

**铁律 3：过长台词先拆句，句间插反应镜头**

如果台词明显过长（一段超过 8 秒的独角戏），应先把它拆成 2-3 个短句，然后**句与句之间**穿插对方的无台词反应镜头。拆句位置必须选自然断点（。！？），绝不能从句子中间断开。

```
✅ 拆句 + 穿插反应：
A（沉声）：十年前，你父亲死在这座山上。
△ 【反应镜头】B 瞳孔微缩，指尖轻轻一颤
A（沉声）：我找了你十年，没想到你竟然变成了他。
△ 【反应镜头】B 偏头避开视线，喉结滚动
A（咬字）：今天，该清算了。

❌ 不拆句直接糊成一段：
A（沉声）：十年前你父亲死在这座山上我找了你十年没想到你竟然变成了他今天该清算了。
→ 15 秒里只有一个人说话，画面僵死
```

### 15秒情绪弧线

- **开场 (0-3s)**：建立情感/氛围
- **发展 (3-9s)**：紧张感上升或情节发展
- **高潮 (9-12s)**：情感高峰或关键揭示
- **释放 (12-15s)**：解决或悬念

### 超长视频（>15秒）：分段拼接

| 总时长 | 分段方案 |
|--------|---------|
| 16-30秒 | 2段（首段15秒+延长段） |
| 31-45秒 | 3段 |
| >60秒 | 拆分为独立场景，剪辑软件拼接 |

**衔接规则**：每段 12-15s 的画面描述 = 下一段 0-3s 的画面描述，文字应保持完全一致，用于后期精确拼接。

### 分集分段实战规范

**15s 基准拆分**：所有文戏以 15s 为一段。

**动作戏/关键剧情单独成段**：打斗、对峙、关键冲突、重要道具亮相、角色崩溃等戏剧高潮必须单独拆成一个完整片段，不与前后段落合并。即使不足 15s 也独立成段。

**纯对话节奏宽松**：文戏每段 5-6 个镜头，每个对白镜头 2.5-3.5 秒。长台词必须断句穿插反应镜头，节奏宁松勿紧——方便初剪时做减法（去掉多余帧比补拍容易得多）。

### 15s 拆分铁律：炸弹结尾，反应给下段（EP8 实战验证）

这是 15s 短剧拆分最核心的原则，来自实战踩坑。原混剪思路把"台词 + 反应"塞进同一个 15s，结果台词说完就超时，反应镜头完全没时间。

**正确做法：**
- 本段结尾落在最强台词/关键动作（炸弹）
- 下一段开头承接反应/情绪消化/围观群众反应
- 每段最多 **2 个情绪节拍**（如「冲突 → 反应」或「出招 → 接招」）

```
✅ 正确拆分：
Seg N: A 说出 37 字长句污蔑 → 话音落地，气氛凝固（炸弹收尾）
Seg N+1: 全场哗然，宾客不怀好意打量 → 被污蔑者强装镇定 → 开始反击

❌ 错误合并：
Seg N: A 污蔑 → 全场哗然 → 被污蔑者镇定 → 反击
→ 15s 塞 4 个节拍，必然超载
```

**动作戏/关键剧情单独成段**：打斗、对峙、关键冲突、重要道具亮相、角色崩溃等戏剧高潮必须单独拆成一个完整片段，不与前后段落合并。即使不足 15s 也独立成段。

### 剧本校对三查（EP8 实战总结）

分镜写完、注入面板前，必须逐句对照原始剧本检查三类错误：

| 类型 | 示例 | 说明 |
|------|------|------|
| **错字** | 醉酒楼 → 醉仙楼 | 地名/人名/专有名词拼写 |
| **多余字** | 我不仅**我**名声尽毁 | 打字时多敲的重复字 |
| **别字** | 戏**虐** → 戏**谑** | 形近/音近误用 |

**检查流程：**
1. 原始剧本 + 分镜文件左右对照，逐行核对每句台词
2. 检查出场人物/场景/道具/时间是否与剧本一致
3. 确保没有"无中生有"的台词或剧情

### 视觉一致性

- 所有资产提示词保持一致艺术风格
- 每个角色使用独特视觉标记（颜色、轮廓、配件）
- 场景描述中参考光线和一天中的时间

## 完整技术模板（必须包含所有标签）

每个 Seedance 提示词必须包含以下固定标签，不可省略任何一项。

**注意：** `【画幅】竖屏 9:16` 和独立的 `15秒，9:16竖屏` 行在面板注入时**不需要**写入——面板的时间轴和画幅选择器已经覆盖这些参数，重复写入只会浪费字数。

```
写实风格古装正剧，极致画质，高对比度光影，电影运镜         ← 风格+画质合并首行
【镜头】ARRI MINI拍摄，中焦段85mm，大光圈f/1.2浅景深，电影写实人像
【环境】[scene chip] [具体位置+时间]
【布光】[primary light source + direction + fill]
【光影】暖色柔光，灯笼辉映
【色调】暖金色调低对比
【特效】背景虚化散景，胶片颗粒感
【运镜】手持拍摄，自然呼吸感运镜，[scene-specific movement]
【构图】非对称构图，禁止越轴
【表演】动作自然流畅，加强眉眼鼻表演，每个镜头人物必须有眨眼动作，不生成多个相同角色
【音频】无字幕，无BGM，全片不生成任何文字——仅环境音（[具体环境音]）

【场景】[scene chip]
【角色】[role chip 1] [role chip 2] [role chip 3] ...
【站位】[一句初始站位描述，描述段落开场的空间关系]

0-3s：【段落标题】景别——[具体画面+动作+对话]
[role chip][audio chip]（情绪）：台词内容。
△ 动作描述。
```

**新增标签说明：**

| 标签 | 格式 | 说明 |
|------|------|------|
| 【场景】 | `[场景chip]` | 引用平台场景资产 |
| 【角色】 | `[角色chip1] [角色chip2] ...` | 引用本段出场的所有角色资产 |
| 【站位】 | 一句初始站位描述 | 描述段落开场的空间关系，后续动作自然展开 |
| 【音频】 | `无字幕，无BGM，全片不生成任何文字——仅环境音（...）` | 明确反相约束 + 指定环境音 |

**注入面板时的 chip 语法：** 分镜文件使用 `[角色名]` 占位符（如 `[傅临琛-贵族常服]`、`[音频-林彩星]`），注入时由浏览器自动化脚本替换为原生 chip HTML。`【场景】`和`【角色】`标签中的 chip 必须使用平台原生 `data-id`，不可用占位 ID。

### 古装剧默认值

| 标签 | 默认值 |
|------|--------|
| 【镜头】 | 85mm，f/1.2 |
| 【光影】 | 暖色柔光，灯笼辉映 |
| 【色调】 | 暖金色调低对比 |
| 【特效】 | 背景虚化散景，胶片颗粒感 |

### 都市短剧默认值

| 标签 | 默认值 |
|------|--------|
| 【镜头】 | 50mm，f/2.8 |
| 【光影】 | 侧顺光为主，环境反射补亮暗部 |
| 【色调】 | 自然日光色温微暖，低对比 |
| 【特效】 | 轻微虚化散景+胶片颗粒，保留毛孔质感 |
| 【构图】 | 非对称构图，禁止越轴 |

### 剧本保真规则

1. **每句台词必须出现且仅一次** — 不可删除、不可重复、不可拆分到两个片段都出现
2. **△ 方向标注是强制节奏点**，不是可选描述
3. **润色扩写（反应镜头、动作细节）可加，但不能替换或删减剧本内容**
4. **长台词分割点选自然停顿**（。！？），不要从句子中间断开

## 质量保证（Checklist）

最终确定前逐项验证：

- ✅ 每个镜头以 `△ ` 开头，运镜使用具体术语
- ✅ 对白正确标注（os/vo/情绪）
- ✅ 包含感官细节（视觉+听觉+触觉）
- ✅ 每集有情绪弧线（开场→发展→高潮→释放）
- ✅ 每段衔接自然，12-15s 与下一段 0-3s 画面一致
- ✅ 所有 @图片X 在资产列表中有对应ID
- ✅ 时间轴覆盖完整15秒，相机运动可行且逻辑顺序
- ✅ 所有角色引用同一套资产编号，不混用
- ✅ 技术标签全部到位：【场景】【角色】【站位】已添加，无反相提示词遗漏
- ✅ 镜头数 5-6 个，配角不超过 1 镜，主角屏占比 ~80%
- ✅ 剧本每句台词出现且仅一次，无跨段重复
- ✅ 15s 拆分遵循「炸弹结尾，反应给下段」原则
- ✅ 分镜已逐句对照剧本校对（错字/多余字/别字）
- ✅ 注入格式正确（`<br>` 换行，无 `【画幅】` 和 `15秒，9:16竖屏` 冗余行）

## 常见陷阱

1. **敏感词**：Seedance 可能拒绝某些术语，使用替代措辞避免
2. **过长提示词**（300+字）：可能导致指令不一致，优先清晰度
3. **缺少连续性**：每段最后场景状态应与下一段开场匹配，不要出现跳跃
4. **模糊镜头语言**：不用"镜头移动"，用"推镜头"、"环绕镜头"
5. **缺乏感官细节**：脚本应唤起视觉+声音+温度+质感
6. **平淡情感弧线**：每集需要情绪进展，不只是情节点推移
7. **风格不一致**：所有资产应用相同视觉风格前缀
8. **动作描述太急**：越慢越稳，避免"夸张""高速""剧烈扭动"
9. **主观抽象词**："漂亮""帅气""很酷" AI无法理解
10. **夜晚出白天**：检查布光方案是否暗示了太阳光
11. **缺少技术标签**：必须包含全部 11 个固定标签（风格/画幅/镜头/环境/布光/光影/色调/特效/运镜/构图/表演），缺一个都可能出问题
12. **忘了反相提示词**：每次必须带"禁止越轴""不生成多个相同角色""眨眼动作""不生成文字"
13. **配角镜头过多**：短剧不是群像剧，配角每段最多 1 个镜头
14. **乒乓切镜头**：不要 A-B-A-B-A-B 来回切换，每隔 4-6 秒插入反应镜头打破单调
15. **台词跨段重复**：分割长台词时检查相邻片段，确保没有相同的句子出现在两个片段
16. **在 A 说话中途切镜头给 B**：AI 不理解对话节奏，切镜头必须等 A 把当前这句话**说完**（。！？之后）再切。一句话说到一半就切 = 嘴型错乱 + 节奏崩坏
17. **画面里出现两个一模一样的角色**：即使反相提示词写了，模型仍有概率触发克隆人。遇到时检查：① 反相提示词是否遗漏 ② 是否引用了多个不同角色但 AI 把它们画成同一张脸
18. **15s 段塞太多节拍**：不要试图在一段里塞「台词→反应→再台词→再反应」四个节拍。实测 37 字台词就吃满 15s，反应镜头必须扔给下一段开头
19. **注入用 `<p>` 标签**：Seedance 编辑器只用 `<br>` 换行，`<p>` 会导致段间距异常和内容被合并乱码

## 浏览器注入规范（EP8 实战总结）

将分镜内容注入 Seedance 面板时，需遵守以下格式规范：

### 换行格式

编辑器使用 `<br>` 扁平换行，**禁止使用 `<p>` 段落标签**。`<p>` 会产生多余段间距，且内容被面板重新解析时会合并成乱码。

```
✅ editor.innerHTML = lines.join('<br>');
❌ editor.innerHTML = lines.map(l => '<p>' + l + '</p>').join('');
```

### Chip 引用语法

分镜文件中使用 `[角色名]` 占位符（如 `[傅临琛-贵族常服]`、`[音频-林彩星]`），注入时由脚本替换为平台原生 chip HTML。

**Chip 提取工作流（最高效方式）：**
1. 用户列出本段需要的所有 chip（角色/场景/音频）
2. 用户在面板编辑器 `@` 出所有 chip
3. 脚本用 `evaluate_script` 一次性提取所有 chip 的 `data-id` / `data-url` / `data-asset-id`
4. 用提取到的原生 HTML 构建注入内容

> ⚠️ 不要自己扒 API 或 Vue 组件树找 chip 数据，让用户 @ 是最快的。

### 冗余行删除

面板的时间轴选择器和画幅选择器已覆盖以下参数，**注入时不需要写入**：
- `【画幅】竖屏 9:16`（面板有画幅下拉框）
- 独立的 `15秒，9:16竖屏`（面板有时长滑块）

这些行在分镜文件中仅作参考标注，注入脚本应跳过。
